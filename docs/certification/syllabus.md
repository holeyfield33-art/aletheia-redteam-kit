# Certified AI Red Team Operator (CARTO) - Exam Blueprint

## Domain 1: AI Security Fundamentals

- Explain the core components of LLM and agent security architectures.
- Distinguish prompt, tool, memory, and workflow trust boundaries.
- Identify common failure modes in model serving, orchestration, and guardrail layers.
- Interpret audit artifacts, receipts, and decision logs during an engagement.

## Domain 2: Attack Techniques

- Classify jailbreak, prompt injection, exfiltration, encoding, and multi-turn attack patterns.
- Select appropriate payloads for single-turn, chained, and staged adversarial tests.
- Evaluate how attack intent maps to severity, impact, and expected enforcement outcomes.
- Compare direct, obfuscated, and blended attack strategies against modern AI systems.

## Domain 3: Tooling and Automation

- Operate CLI and dashboard workflows for API, website, repo, and combined audits.
- Build and extend payload catalogs, threat feeds, and mutation inputs for repeatable testing.
- Interpret run summaries, gate evaluations, and normalized command-center artifacts.
- Automate red-team execution with reproducible environments and artifact pipelines.

## Domain 4: Defense Bypass and Evasion

- Analyze block responses to identify likely guardrail classes and bypass opportunities.
- Apply payload mutation, cloaking, and escalation heuristics to adaptive test loops.
- Assess how encoding, visual rendering, and indirect tool usage affect detection efficacy.
- Measure bypass success while preserving clean-room and non-destructive testing boundaries.

## Domain 5: Reporting and Ethics

- Produce concise findings with evidence, severity, exploitability, and remediation guidance.
- Separate benign control validation from offensive findings in final reporting.
- Explain legal, contractual, and policy constraints for authorized AI red teaming.
- Document operator safety practices, disclosure paths, and evidence handling requirements.
